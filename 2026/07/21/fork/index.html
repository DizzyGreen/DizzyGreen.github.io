fork

fork() 是 Unix/Linux 系统中用于创建进程的核心系统调用（非 C++ 标准库，属于 POSIX 标准）。它会从当前进程（父进程）复制出一个几乎完全相同的子进程

头文件与原型

  • 返回值：
    • 在父进程中：返回新创建子进程的 PID
    • 在子进程中：返回 0
    • 失败时：返回 -1

工作原理（写时拷贝）

  • fork 刚调用时，子进程拷贝父进程的页表，但物理内存与父进程共享（只读）
  • 当任一进程尝试写入内存时，才会触发“写时拷贝（COW）”，真正复制被修改的页面
  • 这种机制使得 fork 非常高效，避免了不必要的内存复

用法示例

与 exec 族函数配合

fork 通常与 exec 一起使用：

  • fork 复制进程，子进程拥有独立的地址空间
  • 子进程调用 exec替换自身映像，执行新程序

需要避免的陷阱

  • 孤儿进程：父进程先于子进程结束，子进程会被 init（PID=1）收养
  • 僵尸进程：子进程结束但父进程未调用 wait() 回收其状态信息，会占用进程表资源。父进程需用 wait() 或 waitpid() 处理
  • 文件描述符共享：fork 后，父子进程共享已打开的文件描述符（如 socket），注意关闭不必要的端以避免混乱
  • 多线程与 fork：只有调用 fork 的线程会被复制，其他线程会消失，极易导致死锁。多线程程序中建议用 posix_spawn 或 vfork

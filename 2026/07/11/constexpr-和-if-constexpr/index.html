constexpr 和 if constexpr

constexpr 是 C++11 引入的一个关键字，并在后续标准（C++14, C++17, C++20）中不断增强了其能力。它的核心目标是允许在编译时计算表达式或函数的值，从而将运行时的计算压力转移到编译期

编译时常量

在 constexpr 出现之前，我们使用 const 来定义常量。但 const 并不总是编译时常量

constexpr 则明确告诉编译器：这个变量或函数必须在编译时就能计算出结果

constexpr 变量

用于声明变量的值是编译时常量

关键点：

  • constexpr 变量必须在声明时初始化
  • 初始化的表达式必须是常量表达式（即它的值在编译时可知）

constexpr 函数

用于声明函数可以在编译时被求值（如果传入的参数是编译时常量）

C++11 下的 constexpr 函数

  • 函数体必须只有一个 return 语句（不能有变量定义、循环、if 等）
  • 返回类型必须是字面类型（通常是基本类型、自定义类型需要满足特定条件）

C++14 及以后下的 constexpr 函数

  • 函数体内可以有多条语句
  • 可以有局部变量（但不能是 static 或 thread_local）
  • 可以有 if, switch, for, while 等控制流语句

constexpr 函数就像一个“双重身份”的函数

  • 当传入的参数是编译时常量时，它会在编译期被计算，结果直接替换到代码中
  • 当传入的参数是运行时常量时，它就像一个普通的函数一样在运行时被调用

这使得你可以写一份代码，同时在编译时和运行时使用

constexpr 与自定义类型

constexpr 也可以用于自定义类型，这允许我们在编译时创建和操作对象

  • constexpr 构造函数：用于在编译时初始化对象
  • constexpr 成员函数：可以在编译时调用

要求：

  • 构造函数体必须为空
  • 必须使用常量表达式来初始化所有成员变量

if constexpr (C++17)

C++17 引入了 if constexpr，它允许在编译时进行条件判断，并丢弃不满足条件的分支代码

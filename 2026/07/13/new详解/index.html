new 详解

new 和 malloc 的区别

  • 本质：new 是 运算符（可重载），malloc 是 函数（不可重载）
  • 返回类型：new 返回 类型安全 的指针（T*），无需强转；malloc 返回 void*，必须强转
  • 大小计算：new 由编译器自动计算，malloc 需手动传字节数
  • 构造/析构：new 会调用构造函数，delete 会调用析构函数；malloc/free 不会
  • 异常处理：new 失败默认抛出 std::bad_alloc 异常；malloc 失败返回 NULL

nothrow new

nothrow new 是 C++ 提供的一种内存分配失败时不抛异常的 new 使用方式

nothrow new 只保证分配内存不抛异常。但如果对象构造函数内部抛异常，程序依然会按正常异常机制传播，此时不会返回 nullptr

::operator new

operator new 是C++中负责内存分配底层机制的全局函数，与之配套的是::operator delete

new 运算符 = operator new 函数（分配内存） + 构造函数调用（构造对象）

当你写下 new T() 时，编译器做了两件事：

  1. 调用 operator new(sizeof(T)) 分配原始内存（这一步和 malloc 类似，但失败抛异常）
  2. 在该内存上调用 T 的构造函数(placement new)

而如果你直接调用 operator new（如 ::operator new(sizeof(T))），你得到的只是未初始化的原始内存

placement new

placement new 是 C++ 中一种特殊的 new 表达式，它在已分配的内存上构造对象，而不分配新内存

要理解 placement new，首先要明白标准的 new 操作符实际上做了两件事：

  1. 分配内存：调用 operator new 函数，在堆上分配足够大小的内存
  2. 构造对象：在刚刚分配好的那块内存地址上，调用对象的构造函数，初始化对象

Placement new 的本质就是只完成上述第二步工作：在已分配好的内存上构造对象。 它不负责内存分配，只负责对象构造

语法：

  • address： 一个指针，指向一块已经存在的、足够容纳 Type 类型对象且对齐正确的内存
  • Type： 要构造的对象的类型
  • arguments： （可选）传递给 Type 的构造函数的参数

必须手动调用析构函数

这是使用 placement new 最重要也是最容易忘记的一点

  • 标准 delete 操作符会先调用析构函数，再调用 operator delete 释放内存
  • 对于 placement new 创建的对象，内存不是你通过 new 分配的，所以你不能用 delete
  • 你必须显式地直接调用析构函数来结束对象的生命周期

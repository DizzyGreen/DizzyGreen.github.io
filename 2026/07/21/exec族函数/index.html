exec族函数

Unix/Linux 中“加载并执行新程序”的核心机制

exec 族函数不创建新进程，而是用新程序完整替换当前进程的代码段、数据段、堆和栈。替换后，进程 ID（PID）保持不变，但执行内容彻底改变。调用成功则不返回；若返回，必定是执行失败（返回 -1）

函数家族

exec 变体

命名规律

函数名由三部分组成：exec + [l/v] + [p] + [e]

后缀 含义 说明
l (list) 参数以列表形式逐个传递 如 execl("/bin/ls", "ls", "-l", NULL)
v (vector) 参数以**字符串数组（vector）**传递 如 execv("/bin/ls", my_argv)
p (PATH) 自动在 PATH 环境变量中搜索可执行文件 如 execlp("ls", "ls", NULL)，无需写绝对路径
e (environment) 可自定义环境变量数组，不继承父进程环境 如 execle("/bin/ls", "ls", NULL, my_env)

组合规则：

  • 必须选 l 或 v（二选一）
  • 可选 p（搜索 PATH）
  • 可选 e（自定义环境）

详细参数说明

  • path/file：要执行的程序路径或文件名（带 p 时可用文件名）
  • arg / argv：传递给新程序的命令行参数。第一个参数惯例是程序名，最后必须以 NULL 结尾（对于 l 系列
  • envp：自定义环境变量数组，格式如 {"HOME=/root", "PATH=/bin", NULL}

用法示例

1. execlp（带 PATH 搜索）

2. execv（参数动态构建时用）

3. execle（自定义环境变量）

与 system() / fork() 的关系

注意事项

  1. exec 后面的代码不执行，除非失败，所以通常紧接 exit(1)
  2. 多线程程序中慎用：exec 会销毁所有线程，只保留调用线程
  3. 信号处理：被 exec 替换后，自定义信号处理函数会恢复为默认行为（SIG_DFL），但被忽略的信号（SIG_IGN）会保持
  4. 环境变量继承：不带 e 时，子进程继承父进程的全部环境变量

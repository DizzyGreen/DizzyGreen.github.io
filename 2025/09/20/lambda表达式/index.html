1. Lambda 表达式的基本结构

一个完整的 Lambda 表达式语法如下

  1. 捕获子句 (Capture clause) []
    • 这是 Lambda 表达式的开头，用于指定如何捕获外部变量（即在 Lambda 函数体外定义的变量）。
    • 它决定了 Lambda 函数体内可以使用哪些外部变量，以及使用的是副本还是引用。
  2. 参数列表 (Parameters) ()
    • 和普通函数一样，用于接收传入的参数。
    • 如果不需要参数，可以省略 ()（在 C++23 之前，无参时括号不可省略，但从 C++20/23 开始，在某些简单情况下可以省略，为了清晰和兼容性，通常建议保留）。
  3. 返回类型 (Return type) -> type
    • 显式指定 Lambda 表达式的返回类型。
    • 如果忽略返回类型，lambda根据函数体中的代码推断出返回类型。如果函数体只是一个return语句，则返回类型从返回的表达式的类型推断而来。如果 lambda的函数体包含任何单一return 语句之外的内容，且未指定返回类型，则返回void。
  4. 函数体 (Body) {}
    • 和普通函数一样，包含要执行的代码。

2. 捕获子句 (Capture Clause) 的详细用法

  • [] 不捕获任何外部变量。

  • [=] 以值的方式捕获所有外部变量。

    • 在 Lambda 体内使用这些变量时，使用的是它们在 Lambda定义时的副本。

  • [&] 以引用的方式捕获所有外部变量。

    • 在 Lambda 体内使用这些变量时，使用的是它们的引用，修改它们会影响外部变量。

  • [var] 显式地以值的方式捕获特定变量 var。

  • [&var] 显式地以引用的方式捕获特定变量 var。

  • 混合捕获

    • 可以混合使用值和引用捕获，也可以混合显式和隐式捕获（但隐式捕获 [=] 或 [&] 必须放在前面）。

  • [this] 捕获当前的 this 指针

    • 用于在类的成员函数中定义 Lambda，使其可以访问该类的成员变量和函数。
    • 本质上是以值的方式捕获了 this 指针，然后通过 this->member 来访问成员。

  • [*this] (C++17) 以值方式捕获当前对象

    • 生成当前对象的一个副本，Lambda 内部的操作不会影响原对象。

3. Lambda 作为函数参数（与 STL 算法配合）

这是 Lambda 最常用的场景，使得 STL 算法的调用非常简洁。

4. mutable 关键字

默认情况下，以值方式 [=] 捕获的变量在 Lambda 体内是 只读的（它们被视为 const）。如果需要在 Lambda 内部修改这些副本的值，并且不希望影响外部变量，需要在参数列表后加上 mutable 关键字。

注意：mutable 允许修改的是 Lambda 对象内部存储的副本，而不是原始外部变量。

5.Lambda 的类型

Lambda 函数的类型是唯一的、未命名的、由编译器自动生成的闭包类型。你没有办法在代码中直接写出它的类型名。每个 Lambda 表达式都会在编译时生成一个全新的、独一无二的类类型。即使两个 Lambda 在代码上看起来完全一样，它们的类型也是不同的。

如何获取和使用 Lambda 的类型？

方法一：使用 auto

这是定义和存储 Lambda 表达式最直接、最简洁的方式。

方法二：使用 std::function

这是将 Lambda 作为参数传递或存储起来的最佳方式。

condition_variable

std::condition_variable 是 C++ 标准库 <condition_variable> 头文件中提供的一个同步原语，用于在线程间进行通信。它允许一个或多个线程等待某个条件成立，直到另一个线程通知它们条件可能已经发生变化

1.核心思想：等待（Wait）与通知（Notify）

condition_variable 通常与一个互斥锁（std::mutex） 和一个共享条件（通常是布尔变量或状态变量） 一起使用

  • 等待方（Waiting Thread）:
    1. 获取一个互斥锁（通常是 std::unique_lock<std::mutex>）
    2. 检查条件是否已经满足。如果满足，则继续执行
    3. 如果条件不满足，则调用 wait() 方法。这个方法会原子地释放互斥锁并使线程进入阻塞状态
    4. 当被其他线程通知后，线程被唤醒，并自动重新获取互斥锁
    5. 由于条件可能在被通知和获取锁之间再次发生变化（这被称为“虚假唤醒”），所以被唤醒后必须再次检查条件。这通常在一个循环中进行
  • 通知方（Notifying Thread）:
    1. 在修改共享条件之前，获取相同的互斥锁
    2. 修改条件
    3. 调用 notify_one() 来唤醒一个正在等待的线程，或者调用 notify_all() 来唤醒所有正在等待的线程

4. 主要成员函数

wait

  1. void wait(std::unique_lock<std::mutex>& lock)

    • 当前线程进入等待状态。
    • 原子操作：调用时，它会释放 lock 并阻塞线程。
    • 被通知唤醒后，它会重新获取 lock，然后 wait 调用返回。
    • 必须在循环中检查条件，因为可能有“虚假唤醒”（即线程在没有被通知的情况下被唤醒）。

  2. void wait(std::unique_lock<std::mutex>& lock, Predicate pred) （推荐用法）

    • pred 是一个可调用对象（如函数、Lambda），返回 bool 值。

    • 只有当 pred() 返回 false 时，线程才会等待；被唤醒后，它会再次检查 pred()。

    •      template <class _Predicate>
     void wait(unique_lock<mutex>& _Lck, _Predicate _Pred) { // wait for signal and test predicate
         while (!_Pred()) {
             wait(_Lck);
         }
     }

7. 重要注意事项

  1. std::unique_lock 是必须的：condition_variable::wait 只能与 std::unique_lock<std::mutex> 一起使用，因为它需要在等待时灵活地解锁和重新加锁。std::lock_guard 没有这个能力
  2. 虚假唤醒（Spurious Wakeup）：即使没有线程调用 notify，等待的线程也可能被唤醒。这是某些操作系统实现允许的行为。因此，必须在循环中检查条件是否真正满足。使用带谓词的 wait 是防御虚假唤醒的最佳实践
  3. 丢失唤醒（Lost Wakeup）：如果在消费者调用 wait() 之前，生产者就调用了 notify_one()，那么这个通知可能会丢失，导致消费者永远等待。使用带条件的 wait 可以避免这个问题，因为即使通知丢失，消费者在第一次检查条件时如果发现条件满足（队列不空），就不会进入等待
  4. condition_variable 的生命周期：确保条件变量的生命周期长于所有使用它的线程。不要在还有线程可能等待它时销毁它
